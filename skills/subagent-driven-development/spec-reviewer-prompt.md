# 规格合规审查者提示模板

派遣规格合规审查子代理时使用此模板。

**目的：** 验证实现者构建了请求的内容（不多不少）

```
Task tool (general-purpose):
  description: "审查任务 N 的规格合规"
  prompt: |
    你正在审查一个实现是否匹配其规格。

    ## 请求的内容

    [任务需求的完整文本]

    ## 实现者声称构建的内容

    [来自实现者的报告]

    ## 关键：不要相信报告

    实现者完成得太可疑地快。他们的报告可能不完整、
    不准确或过于乐观。你必须独立验证一切。

    **不要：**
    - 相信他们实现的说辞
    - 相信他们关于完整性的声称
    - 接受他们对需求的解读

    **要：**
    - 阅读他们编写的实际代码
    - 逐行将实际实现与需求比较
    - 检查他们声称实现但缺失的部分
    - 寻找他们没有提到的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失的需求：**
    - 他们实现了请求的所有内容吗？
    - 有他们跳过或遗漏的需求吗？
    - 他们声称某些东西工作但实际没实现吗？

    **额外/不需要的工作：**
    - 他们构建了未请求的东西吗？
    - 他们过度工程或添加了不必要的功能吗？
    - 他们添加了规格中没有的"有则更好"吗？

    **误解：**
    - 他们对需求的解读与意图不同吗？
    - 他们解决了错误的问题吗？
    - 他们实现了正确的功能但方式错误吗？

    **通过阅读代码验证，不是通过相信报告。**

    报告：
    - ✅ 规格合规（如果在代码检查后一切匹配）
    - ❌ 发现问题：[具体列出缺失或额外的内容，带 file:line 引用]
```
